# Distribution

This document defines how Envarly release artifacts are produced, described, and checked before a public release.

## Release Artifacts

Each tagged GitHub Release should include:

| Artifact | Audience | Notes |
|---|---|---|
| NSIS `.exe` installer | Recommended direct download for most users | Branded installer, familiar double-click install flow. Use this for README and landing page download copy unless release testing says otherwise. |
| WiX `.msi` installer | Package managers, managed installs, and automation | Branded installer. Prefer this as the first WinGet candidate when install, upgrade, uninstall, and silent modes pass release testing. |
| Portable `.zip` | Manual testing and no-installer use | Contains the release executable only. Does not create shortcuts or uninstall entries. |
| `SHA256SUMS.txt` | Verification and package manager submissions | Generated by the release workflow from the uploaded installer artifacts. |

## Download Guidance

Use this copy when describing downloads:

- **Recommended:** download the `.exe` installer.
- **Managed install:** use the `.msi` installer.
- **Portable:** download the `.zip` archive.

If MSI testing shows a better direct-user experience for a specific release, update the release notes and landing-page copy for that release before publishing.

## Release Checklist

Before tagging a release:

- Confirm `package.json`, `src-tauri/tauri.conf.json`, and `src-tauri/Cargo.toml` have the same version.
- Confirm the release workflow still uploads NSIS `.exe`, WiX `.msi`, portable `.zip`, and `SHA256SUMS.txt`.
- Build locally if the change touches installer configuration:

```powershell
npm run tauri build
```

After the GitHub Release workflow finishes, download the artifacts from the draft or published release and check:

| Check | EXE | MSI | ZIP |
|---|---:|---:|---:|
| Fresh install opens Envarly | Required | Required | Required |
| Upgrade from previous release keeps app data | Required | Required | N/A |
| Uninstall removes the app | Required | Required | N/A |
| PATH cleanup works after uninstall | Required | Required | N/A |
| Silent install works | Required | Required | N/A |
| Silent uninstall works | Required | Required | N/A |
| `SHA256SUMS.txt` includes the artifact | Required | Required | Required |

Suggested silent install checks:

```powershell
# NSIS installer
.\Envarly_<version>_x64-setup.exe /S

# MSI installer
msiexec /i .\Envarly_<version>_x64_en-US.msi /qn /norestart
```

Suggested silent uninstall checks:

```powershell
# NSIS uninstall, from the installed directory
& "$env:LOCALAPPDATA\Envarly\uninstall.exe" /S

# MSI uninstall
msiexec /x .\Envarly_<version>_x64_en-US.msi /qn /norestart
```

Adjust paths and filenames to match the artifacts produced by the release workflow.

## Package Manager Notes

For WinGet:

- Prefer the MSI artifact first if the release checklist passes for install, upgrade, uninstall, and silent modes.
- Use the GitHub Release artifact URL and the matching SHA256 from `SHA256SUMS.txt`.
- Fall back to the EXE installer only if MSI behavior is worse for that release.

For Scoop:

- Use the GitHub Release artifact URL and the matching SHA256 from `SHA256SUMS.txt`.
- The portable ZIP is the best candidate if Scoop install behavior should avoid a traditional installer.

## Release Notes Template

Include a short download section in each release:

```markdown
## Downloads

- Recommended: `Envarly_<version>_x64-setup.exe`
- Managed install: `Envarly_<version>_x64_en-US.msi`
- Portable: `Envarly_<version>_x64-portable.zip`
- Checksums: `SHA256SUMS.txt`
```
